3. **`src/entrypoints/background.ts`** -- Replace placeholder with full wallet message handler:

   **Imports:** Import crypto functions from `@/features/wallet/crypto`. Import types from `@/features/wallet/types`.

   **Storage helpers:**
   - `saveVault(blob: VaultBlob)` -> `chrome.storage.local.set({ vault: blob })`
   - `loadVault()` -> `chrome.storage.local.get('vault')` returns VaultBlob | null
   - `cacheSession(data: { seed: string; accounts: DerivedAccount[] })` -> `chrome.storage.session.set({ walletSession: data })`
     - **Hex-encode seed before storing** (chrome.storage serializes to JSON; Uint8Array becomes broken object)
   - `getSession()` -> `chrome.storage.session.get('walletSession')` returns session data or null
   - `clearSession()` -> `chrome.storage.session.remove('walletSession')`

   **Message handler:** `handleWalletMessage(msg: WalletMessage): Promise<WalletResponse>`

   **`wallet:create`:**
   1. Generate mnemonic (using msg.strength or default 128)
   2. Convert mnemonic to seed
   3. Derive account at index 0
   4. Create VaultPlaintext `{ mnemonic, createdAt: Date.now() }`
   5. Encrypt vault with msg.password
   6. Save encrypted vault to chrome.storage.local
   7. Cache seed (hex-encoded) + account 0 in chrome.storage.session
   8. Return `{ type: 'wallet:created', address, mnemonic }` -- this is the ONLY response that includes mnemonic

   **`wallet:import`:**
   1. Validate mnemonic with isValidMnemonic. If invalid, return error.
   2. Convert mnemonic to seed
   3. Derive account at index 0
   4. Create VaultPlaintext, encrypt, save to storage.local
   5. Cache seed + account 0 in storage.session
   6. Return `{ type: 'wallet:imported', address }` -- NO mnemonic in response

   **`wallet:unlock`:**
   1. Check lockout manager. If locked, return error with remaining time.
   2. Load vault from storage.local. If no vault, return error.
   3. Decrypt vault with password. If fails, record failure in lockout manager, persist lockout state to storage.session, return error.
   4. On success: reset lockout, convert mnemonic to seed, derive account 0, cache in storage.session.
   5. Return `{ type: 'wallet:unlocked', address }`

   **`wallet:lock`:**
   1. Clear storage.session
   2. Return `{ type: 'wallet:locked' }`

   **`wallet:getAccounts`:**
   1. Get session. If no session, return error "Wallet is locked".
   2. Return `{ type: 'wallet:accounts', accounts: session.accounts }`

   **`wallet:deriveAccount`:**
   1. Get session. If no session, return error "Wallet is locked".
   2. Decode seed from hex
   3. Derive account at requested index
   4. Add to session accounts list, update storage.session
   5. Return `{ type: 'wallet:derived', account: { index, address, path } }` -- NO private key

   **Listener registration:**
   ```typescript
   chrome.runtime.onMessage.addListener((msg, _sender, sendResponse) => {
     if (msg.type?.startsWith('wallet:')) {
       handleWalletMessage(msg as WalletMessage).then(sendResponse);
       return true; // async response
     }
   });
   ```

   **On service worker start:** Initialize lockout manager. Try to restore lockout state from storage.session.

   **SECURITY RULES (enforced in code):**
   - NEVER include `privateKey` in any WalletResponse
   - NEVER include `mnemonic` in any response except `wallet:created`
   - NEVER log mnemonic, seed, or private key values
   - ALL error messages are generic -- "Incorrect password", "Wallet is locked", "Invalid mnemonic"
