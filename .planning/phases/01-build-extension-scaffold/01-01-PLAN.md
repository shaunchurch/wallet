2. **`scripts/build.ts`** -- Multi-call esbuild build script per research Pattern 1. Four separate esbuild.build() calls:
   - Background: src/entrypoints/background.ts -> dist/background.js, format ESM
   - Content: src/entrypoints/content.ts -> dist/content.js, format IIFE
   - Inpage: src/entrypoints/inpage.ts -> dist/inpage.js, format IIFE
   - Popup: src/entrypoints/popup.tsx -> dist/popup.js, format ESM, with postcssPlugin, JSX/TSX loader, SVG as dataurl

   Common options: bundle true, target chrome120, define process.env.NODE_ENV. Minify and disable sourcemaps in production. Enable sourcemaps in development.

   After builds: copy public/manifest.json, public/popup.html, and public/icons/ to dist/. Use fs.cpSync.

   Handle --watch flag: if process.argv includes '--watch', use esbuild context.watch() for all builds instead of one-shot build.

   **CRITICAL for deterministic builds (BUILD-04):** Do NOT include Date.now(), __dirname, timestamps, or host-specific values in define. Only static strings like '"production"' and '"0.1.0"'.

3. **`src/entrypoints/background.ts`** -- Minimal service worker placeholder per research example. Log startup, handle onInstalled.

4. **`src/entrypoints/content.ts`** -- Per research Pattern 3. Create script tag, set src to chrome.runtime.getURL('inpage.js'), append to document, remove on load.

5. **`src/entrypoints/inpage.ts`** -- Minimal placeholder. Log that inpage script loaded. Comment that EIP-1193 provider will be added in Phase 5.

6. **`src/entrypoints/popup.tsx`** -- Per research code example. StrictMode, createRoot, MemoryRouter, render App component. Import globals.css.
