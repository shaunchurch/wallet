  <behavior>
    TEST-03 (serialization.test.ts):
    - Use a known private key (e.g., from Ethereum test vectors or deterministic key)
    - Build an EIP-1559 Type 2 transaction with known params (to, value, nonce, gasLimit, maxFeePerGas, maxPriorityFeePerGas, chainId=1)
    - Sign with buildAndSignTransaction
    - Verify the output starts with 0x02 (Type 2 prefix)
    - Verify the signed tx can be decoded back to recover the correct sender address using micro-eth-signer's Transaction.fromHex()
    - Verify that the transaction fields (to, value, nonce, chainId) in the decoded tx match the input params
    - Test with at least 2 different tx param sets to ensure RLP encoding handles varying field sizes
    - **CRITICAL: Known-vector hex comparison** — at least 1 test case must use a published EIP-1559 Type 2 test vector (from ethereum/tests repo, EIP-1559 spec examples, or ethers.js test fixtures) where the expected signed tx hex is hardcoded. Compare the output of buildAndSignTransaction byte-for-byte against this known hex. This catches RLP encoding bugs that roundtrip tests would miss (e.g., field ordering, access list encoding, signature recovery bit). Source the vector from an external reference, NOT from running the implementation.

    TEST-04 (gas.test.ts):
    - Mock rpcCall to return various eth_estimateGas values
    - Test: estimate of 21000 (standard ETH transfer) -> after 20% buffer = 25200 -> below floor -> returns 60000
    - Test: estimate of 50000 -> after 20% buffer = 60000 -> equals floor -> returns 60000
    - Test: estimate of 100000 -> after 20% buffer = 120000 -> above floor -> returns 120000
    - Test: estimate of 0 -> returns 60000 (floor)
    - Assert: no test case ever returns a value < 60000

    TEST-06 (nonce.test.ts):
    - Mock rpcCall for eth_getTransactionCount to return incrementing values
    - Call the nonce fetch logic for two sequential transactions
    - Verify second nonce = first nonce + 1
    - Test: if pending count returns "0x5", nonce should be 5n
    - Test: if pending count returns "0x0", nonce should be 0n
  </behavior>
