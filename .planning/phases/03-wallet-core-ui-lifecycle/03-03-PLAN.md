3. **`src/entrypoints/background.ts`** -- Auto-lock + seed export:

   **Auto-lock alarm system (SEC-08):**

   Constants:
   ```typescript
   const AUTO_LOCK_ALARM = 'auto-lock';
   const DEFAULT_AUTO_LOCK_MINUTES = 15;
   ```

   Top-level alarm listener (MUST be at module top-level to survive SW restart):
   ```typescript
   chrome.alarms.onAlarm.addListener(async (alarm) => {
     if (alarm.name === AUTO_LOCK_ALARM) {
       await clearSession();
       await removePendingCreation();
     }
   });
   ```

   Helper functions:
   ```typescript
   async function resetAutoLockAlarm(): Promise<void> {
     const result = await chrome.storage.local.get('autoLockMinutes');
     const minutes = (result.autoLockMinutes as number) || DEFAULT_AUTO_LOCK_MINUTES;
     await chrome.alarms.clear(AUTO_LOCK_ALARM);
     await chrome.alarms.create(AUTO_LOCK_ALARM, { delayInMinutes: minutes });
   }

   async function clearAutoLockAlarm(): Promise<void> {
     await chrome.alarms.clear(AUTO_LOCK_ALARM);
   }
   ```

   **Integration points (lifecycle events):**
   - After `handleUnlock` success: call `await resetAutoLockAlarm()`
   - After `handleCreate` / `handleImport` success (wallet starts unlocked): call `await resetAutoLockAlarm()`
   - After `handleConfirmSeedPhrase` success: call `await resetAutoLockAlarm()`
   - In `handleLock`: call `await clearAutoLockAlarm()`

   **Inactivity reset (SEC-08 — true inactivity timeout):**
   Add a new message type `wallet:heartbeat` that resets the auto-lock alarm on user interaction.

   In `WalletMessage` union: `| { type: 'wallet:heartbeat' }`
   In `WalletResponse` union: `| { type: 'wallet:heartbeatAck' }`

   Handler: if session exists, call `resetAutoLockAlarm()` and return `wallet:heartbeatAck`. If locked, no-op (return error).

   **Popup side:** In the popup root (App.tsx or store.initialize), register a throttled listener on `document` for `click`, `keydown`, `scroll` events. On any activity, send `wallet:heartbeat` to background. Throttle to max once per 60 seconds to avoid message spam.

   This makes auto-lock a true inactivity timer: the alarm resets on any user interaction, not just lifecycle events. Active users stay unlocked; idle users get locked.

   **Service worker startup with ready-promise gate:**
   Replace the existing fire-and-forget `restoreLockout()` call at the bottom of background.ts with a gated initialization pattern:

   ```typescript
   // Ready promise — gates message handling until init completes
   const ready: Promise<void> = (async () => {
     await restoreLockout();
     // Re-register auto-lock alarm if session exists (handles browser restart)
     const session = await getSession();
     if (session) {
       const alarm = await chrome.alarms.get(AUTO_LOCK_ALARM);
       if (!alarm) {
         await resetAutoLockAlarm();
       }
     }
   })();
   ```

   Then in `handleWalletMessage`, add `await ready;` as the first line inside the try block. This ensures lockout state is restored before any message is processed, fixing the race where popup queries arrive before restoreLockout() completes.

   **New message handlers:**

   Add cases to `handleWalletMessage` switch:

   **`wallet:exportSeedPhrase`** (SEC-09):
   1. Check session exists (wallet must be unlocked). If not, return error "Wallet is locked".
   2. Load vault from storage.local. If no vault, return error.
   3. Decrypt vault with provided password. If fails, record lockout failure, return error "Incorrect password".
   4. On success: reset lockout, return `{ type: 'wallet:seedPhrase', mnemonic: plaintext.mnemonic }`.
   5. **Why separate from unlock:** Clear intent separation. Export is a sensitive action with its own audit trail. Password re-entry is defense-in-depth against unlocked-but-unattended wallets.

   **`wallet:setAutoLockTimeout`** (SET-02):
   1. Validate `msg.minutes` is one of [5, 15, 30, 60]. If not, return error.
   2. Persist to `chrome.storage.local.set({ autoLockMinutes: msg.minutes })`.
   3. If session exists (unlocked), reset alarm with new timeout: `await resetAutoLockAlarm()`.
   4. Return `{ type: 'wallet:settingsSaved' }`.

   **`wallet:getAutoLockTimeout`:**
   1. Read from `chrome.storage.local.get('autoLockMinutes')`.
   2. Return `{ type: 'wallet:autoLockTimeout', minutes: result.autoLockMinutes ?? DEFAULT_AUTO_LOCK_MINUTES }`.
